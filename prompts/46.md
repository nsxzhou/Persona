# 46. 首秀数据诊断

## 提示词内容

```json
{
  "task": "diagnose_performance",
  "metrics": {
    "ctr": "{{ctr}} (点击率)",
    "read_rate_100k": "{{read_rate}} (10万字完读率)"
  },
  "benchmarks": {
    "ctr_pass": "8%+",
    "read_rate_pass": "15%+"
  },
  "analysis_logic": [
    "若CTR低 -> 检查书名、封面、简介前三行",
    "若完读率低 -> 检查黄金三章是否劝退、钩子是否够硬、期待感是否断裂"
  ],
  "output_format": "诊断报告 + 具体修改方案"
}
```

## 使用场景
数据分析。根据首秀成绩诊断问题所在。

## 最佳实践要点
1.  **基准线 (Benchmarks)**：设定明确的合格线（CTR 8%, 完读 15%），作为判断依据。
2.  **逻辑推演**：建立“数据-原因”的诊断逻辑链，快速定位问题。

## 示例输入
填入点击率和完读率。
